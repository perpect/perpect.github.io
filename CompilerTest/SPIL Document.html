SPIL 문서 ver.1.0

1. 빠르게 시작하기

SPIL에 오신 것을 환영합니다! 자동 근무표 생성을 위한 기본적인 개념과 문법을 익히며 예제들을 통해 실무에 적합한 정책들을 생성해봅시다!
이 문서는 우선 교대근무 관리체계의 기본적인 기능들을 익혔다는 전제하에 알려드릴 것이니 잘 모르신다면 교대근무 관리체계 매뉴얼부터 읽고 오시는 걸 추천드립니다.

1.1. 자동 근무 생성 알고리즘 소개

자동 근무 생성은 Simulated Annealing(SA)라는 최적화 알고리즘을 통해 구현되었습니다.
간단히 설명드리자면, SA는 주어진 조건을 만족하는 최적의 결과를 찾기 위해 여러 가지 가능한 해결책을 시도하는 과정입니다.
처음에는 무작위로 여러 후보를 생성하고, 점차 더 나은 결과로 이동하는 방식입니다. 여기서는 사용자가 지정한 정책에 가장 적합한 근무표를 찾기 위해 여러 후보 근무표들을 만들어내고, 그 중 가장 최적의 근무표를 선택하여 최적화합니다.

1.2. 예제 : 야간 근무 후 비번 보장 정책

다음은 간단한 야간 근무 후 비번을 보장하는 정책입니다. 

if (근무 == 야간) {
    if (다음날.근무 != 비번){
        fail;
    }
}

1.3. "셀"의 개념

"셀"은 하나의 근무 항목을 의미합니다. 예를 들어, 각 근무자와 날짜별로 배치되는 근무를 하나의 셀로 간주할 수 있습니다.
각 셀은 날짜와 근무자 정보를 포함하고 있으며, 근무 유형(예: 주간, 야간, 비번) 등의 규칙이 이 셀을 기준으로 적용됩니다.
예를 들어, "2025년 5월 1일"에 "야간근무"가 배치된 셀이 있을 수 있으며, 이 셀은 "다음날" 셀과 비교되어 규칙이 적용됩니다.

1.4. 예제를 통해 살펴본 작동 방식

SPIL에서는 정책을 적용하거나 평가할 때, 셀 단위로 평가가 이루어집니다. 예를 들어, **"다음날"**이라는 조건은 해당 셀의 오른쪽에 있는 셀을 가리킵니다.
예제에 있는 정책은 다음과 같이 근무표를 평가하게 됩니다 :
1. 야간근무가 배치된 셀을 찾습니다. (근무 == 야간근무)
2. 그 다음날 셀(오른쪽 셀)의 근무가 비번이 아닌지 확인합니다. (다음날.근무 != 비번)
3. 만약 비번이 아니면, 해당 후보는 탈락(fail) 처리됩니다.
이 규칙을 통해, 야간근무 후 비번이 아닌 근무가 배치된 후보들은 자동으로 탈락하게 되며, 결과적으로 야간근무 후에는 비번이 보장된 근무표만 살아남게 됩니다.
